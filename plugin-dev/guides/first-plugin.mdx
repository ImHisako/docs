Building a plugin for Illegalcord is straightforward once your environment is ready. This guide walks you through creating **PrefixAdder** — a plugin that prepends a custom text prefix to every message you send. By the end, you'll have a working, configurable plugin running inside Discord.

## Prerequisites

- Completed the Illegalcord [developer environment setup](/plugin-dev/getting-started)
- Illegalcord installed with **Developer Mode** enabled (Settings → Illegalcord → Advanced → Developer Mode)

## Build PrefixAdder

<Steps>
  <Step title="Create the plugin file">
    Create a new folder for your plugin (e.g. `PrefixAdder/`) inside your local plugins directory, then add `index.ts` with the following content:

    ```typescript index.ts
    import { definePlugin, definePluginSettings } from "@illegalcord/api";
    import { Patcher, findByProps } from "@illegalcord/api";

    const settings = definePluginSettings({
      prefix: {
        type: "string",
        description: "Text to prepend to every message",
        default: "[Illegalcord] ",
      },
      enabled: {
        type: "boolean",
        description: "Enable the prefix",
        default: true,
      },
    });

    export default definePlugin({
      name: "PrefixAdder",
      description: "Prepends a custom prefix to every message you send",
      authors: [{ name: "YourName", id: 0n }],
      settings,

      start() {
        const MessageActions = findByProps("sendMessage");
        Patcher.before("PrefixAdder", MessageActions, "sendMessage", (_, args) => {
          const [, message] = args;
          if (settings.store.enabled && message.content) {
            message.content = settings.store.prefix + message.content;
          }
        });
      },

      stop() {
        Patcher.unpatchAll("PrefixAdder");
      },
    });
    ```

    Replace `"YourName"` with your own display name. The `id` field accepts a Discord user snowflake as a `BigInt` — use `0n` as a placeholder while developing locally.
  </Step>

  <Step title="Create manifest.json">
    In the same `PrefixAdder/` folder, add a `manifest.json` file that describes your plugin:

    ```json manifest.json
    {
      "name": "PrefixAdder",
      "description": "Prepends a custom prefix to every message you send",
      "version": "1.0.0",
      "authors": [
        { "name": "YourName" }
      ]
    }
    ```

    Illegalcord reads this file to display your plugin's name and description in the plugin browser. Keep the `version` field in semantic versioning format (`MAJOR.MINOR.PATCH`).
  </Step>

  <Step title="Load the plugin">
    With both files saved, load your plugin into Discord:

    1. Open Discord and navigate to **Settings → Illegalcord → Plugins**.
    2. Click **Load Unpacked**.
    3. Select your `PrefixAdder/` folder.
    4. Toggle the plugin **on** using the switch next to its name.

    Illegalcord will call your plugin's `start()` function immediately after you enable it.
  </Step>

  <Step title="Configure the prefix">
    Now that PrefixAdder is running, customize your prefix:

    1. Click the **gear icon** (⚙️) next to PrefixAdder in the plugin list.
    2. Set your desired prefix text in the **Text to prepend to every message** field.
    3. Open any Discord channel and send a test message.
    4. Confirm the message appears with your prefix prepended.

    <Tip>
      Try a short, recognizable prefix like `[Test] ` during development so you can quickly spot prefixed messages in the chat log.
    </Tip>
  </Step>

  <Step title="Edit and hot-reload">
    Illegalcord watches your plugin files and reloads automatically when it detects a change:

    1. Open `index.ts` in your editor and change the `default` value of `prefix` to something new.
    2. Save the file — Illegalcord detects the change and hot-reloads PrefixAdder.
    3. Send another test message and verify the new prefix is applied.

    You don't need to manually disable and re-enable the plugin after each save. Hot-reload calls `stop()` to remove existing patches, then `start()` to apply the updated code.
  </Step>
</Steps>

<Note>
  Calling `Patcher.unpatchAll("PrefixAdder")` inside `stop()` is essential. It removes the `sendMessage` patch when the plugin is disabled or reloaded, preventing duplicate patches from stacking on top of each other.
</Note>

<Tip>
  Open the Discord DevTools console with **Ctrl+Shift+I** and run the following to inspect your live plugin instance:

  ```javascript
  Illegalcord.Plugins.getPlugin("PrefixAdder")
  ```

  This returns the plugin object, including its current settings store, so you can confirm values are being read correctly without adding extra logging.
</Tip>

## Next steps
