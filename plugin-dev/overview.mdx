Illegalcord exposes a powerful JavaScript and TypeScript API that lets you reach directly into Discord's client-side internals — patching functions, swapping out React components, reacting to events, and surfacing your own settings UI — all without touching Discord's source code. If you can write JavaScript, you can build a plugin.

## What the plugin system is

The Illegalcord plugin system is a sandboxed runtime that sits on top of Discord's bundled Webpack modules. At load time, Illegalcord injects your plugin code into the client, giving you controlled access to Discord's internal module registry, its React component tree, and its event bus. Plugins are loaded and unloaded dynamically, so users can toggle them on or off without restarting the client.

## What plugins can do

Illegalcord plugins are not limited to cosmetic tweaks. Because they run inside the same JavaScript context as Discord, they can interact with almost every layer of the client.

## Plugin anatomy

Every Illegalcord plugin is built around a single `definePlugin()` call exported from the plugin's entry point. This call tells Illegalcord the plugin's identity and wires up its lifecycle hooks.

```typescript
import { definePlugin } from "@illegalcord/api";

export default definePlugin({
  name: "MyPlugin",
  description: "A short description of what this plugin does.",
  authors: [{ name: "YourName", id: "your-discord-id" }],

  start() {
    // Called when the plugin is enabled.
    // Register patches, event listeners, and settings here.
  },

  stop() {
    // Called when the plugin is disabled.
    // Undo anything you set up in start().
  },
});
```

The three required fields are `name`, `description`, and `authors`. The `start()` and `stop()` lifecycle methods are optional but recommended — `start()` runs when your plugin is activated, and `stop()` runs when it is deactivated, giving you a clean place to set up and tear down side effects.

## Tech stack

| Layer | Technology |
|---|---|
| Language | JavaScript or TypeScript (TypeScript strongly recommended) |
| UI framework | React (same version Discord ships) |
| Module access | Webpack module patching via Illegalcord's `findModule` / `findByProps` APIs |
| Build tooling | esbuild (bundled into the Illegalcord dev toolchain) |

<Info>
  Illegalcord ships TypeScript type definitions for its entire public API. Using TypeScript gives you autocomplete, inline documentation, and compile-time safety — all of which make plugin development significantly faster.
</Info>

## Prerequisites

Before you write your first plugin, make sure you have the following in place:

- **Node.js 18 or later** — required for the build toolchain and package management.
- **Basic JavaScript or TypeScript knowledge** — you should be comfortable with ES modules, arrow functions, and `async/await`.
- **Familiarity with React** — not strictly required for simple patches, but essential for any plugin that adds or modifies UI components.
- **Discord installed with Illegalcord** — you need a running Illegalcord client to load and test your plugin.

<Note>
  You do not need to understand Webpack internals deeply to get started. Illegalcord's helper functions abstract away the low-level module resolution, so you can focus on what your plugin actually does.
</Note>
